# 📄 NpureStudio 実装詳細設計書 (Implementation Ready)

## 1. システム構成
`transformers.js v3 (WebGPU)` を使用し、iPhone 17 Pro の NPU を活用したオンデバイス画像生成フローを実装する。

### 構成ファイル
- `src/features/translator.js`: 日本語→英語翻訳
- `src/features/generator.js`: Stable Diffusion 画像生成
- `src/app.js`: UIイベントと上記モジュールの統合

---

## 2. 翻訳モジュール (`src/features/translator.js`)
**指示:** 軽量な T5 モデルを用いて、日本語入力をプロンプト用の英語に変換する。

- **使用モデル:** `Xenova/t5-small` (Task: `translation_ja_to_en`)
- **主要関数:**
    - `initTranslator(transformers, device)`: パイプラインの初期化。
    - `translate(text)`: 日本語を受け取り、英訳文字列を返す。
    - `disposeTranslator()`: メモリ解放。

---

## 3. 生成モジュール (`src/features/generator.js`)
**指示:** WebGPU バックエンドを使用し、1-4ステップで高速に画像を生成する。

- **使用モデル:** `onnx-community/sd-turbo-onnx`
- **実行設定:**
    - `device`: `webgpu`
    - `dtype`: `fp16` (iPhone NPU 最適化)
- **主要関数:**
    - `initGenerator(transformers)`: `pipeline('text-to-image', ...)` の初期化。
    - `generateImage(prompt, options)`: 
        - `num_inference_steps: 4` 固定。
        - `guidance_scale: 0.0` (Turboモデル用)。
        - 戻り値は `RawImage` オブジェクト。
    - `disposeGenerator()`: 巨大なモデルデータをメモリから破棄。

---

## 4. アプリケーション統合ロジック (`src/app.js`)
**指示:** 既存の `NpureStudio` クラスに、以下の生成フローを組み込む。

1.  **イベントトリガー:** - `tryOnBtn`（または新設の生成ボタン）クリック時に動作。
2.  **実行フロー:**
    - `updateStatus` で「翻訳中...」と表示。
    - `translator.translate(inputField.value)` を実行。
    - `updateStatus` で「画像生成中 (WebGPU)...」と表示。
    - `generator.generateImage(translatedText)` を実行。
    - 得られた `RawImage` を `canvas` に描画。
3.  **エラーハンドリング:**
    - WebGPU 起動失敗時は `wasm` へフォールバックするか、ユーザーに警告を出す。
