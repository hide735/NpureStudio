# NpureStudio 詳細設計書

## 1. 共通部詳細設計

### 1.1 アーキテクチャ概要
- **モジュール構造**: アプリを以下のモジュールに分割
  - `core/`: WebGPU 初期化、Transformers.js 統合
  - `ui/`: UI コンポーネント（キャンバス、ボタン、入力フォーム）
  - `features/`: 各機能（着せ替え、画像生成）の実装
  - `utils/`: ヘルパー関数（画像処理、ファイル操作）
- **データフロー**: ユーザ入力 → UI コンポーネント → 機能モジュール → WebGPU 処理 → 結果表示

### 1.2 WebGPU 統合
- **初期化**: Safari 26+ で WebGPU を検知し、GPUDevice を取得
- **メモリ管理**: iPhone のメモリ制限（最大 1GB）を考慮し、テクスチャとバッファを効率的に管理
- **エラーハンドリング**: GPU 利用不可時のフォールバック（CPU 処理）

### 1.3 Transformers.js 統合
- **バージョン**: v3 を使用
- **モデル管理**: オンデバイスでモデルをキャッシュ、必要に応じてロード
- **パフォーマンス最適化**: モデルを WebGPU で実行、NPU を活用

### 1.4 共通 UI コンポーネント
- **キャンバス**: HTML5 Canvas または WebGL Canvas を使用、タッチ操作対応
- **ファイルアップロード**: ドラッグ&ドロップとファイル選択をサポート
- **プログレスバー**: AI 処理中の進捗表示
- **エラーメッセージ**: 処理失敗時のユーザーフレンドリーな通知

## 2. 自動着せ替え部詳細設計

### 2.1 機能概要
人物写真と衣服写真を入力とし、AI を用いて仮想試着を実現。

### 2.2 ワークフロー
1. **入力受付**: 人物写真と衣服写真をアップロード
2. **セグメンテーション**: SAM (Segment Anything Model) で人物と衣服のマスク生成
3. **特徴抽出**: IP-Adapter で衣服の特徴を抽出
4. **インペイント**: Stable Diffusion Inpainting で衣服を差し替え
5. **出力**: 合成画像を表示

### 2.3 アルゴリズム詳細
- **SAM モデル**: Transformers.js でロード、人物領域と衣服領域を自動検出
- **IP-Adapter**: 衣服画像から特徴ベクトルを生成
- **Inpainting**: マスク領域を条件付き生成、プロンプトでスタイル調整

### 2.4 UI 設計
- **レイアウト**: 2カラム（左: 人物写真、右: 衣服写真）
- **操作**: タッチでマスク微調整、ピンチでズーム
- **ボタン**: 「試着開始」ボタン、結果プレビュー
- **モバイル最適化**: iPhone 画面サイズにフィット、縦向き優先

## 3. 汎用画像生成部詳細設計

### 3.1 機能概要
テキストや画像から新しい画像を生成・編集。

### 3.2 サブ機能詳細

#### 3.2.1 Text-to-Image
- **入力**: テキストプロンプト
- **処理**: SDXL Turbo で高速生成
- **出力**: 生成画像表示

#### 3.2.2 Image-to-Image
- **入力**: 元画像 + プロンプト
- **処理**: スタイル転送または背景変更
- **出力**: 編集後画像

#### 3.2.3 Object Removal
- **入力**: 元画像 + 削除対象指定
- **処理**: インペイントで対象を除去
- **出力**: 修正画像

### 3.3 共通処理
- **モデル**: Stable Diffusion 系を使用
- **パラメータ**: 解像度、ステップ数、CFG Scale を調整可能

### 3.4 UI 設計
- **レイアウト**: 中央キャンバス + サイドパネル（プロンプト入力、設定）
- **操作**: キャンバス上でブラシツールでマスク描画
- **ボタン**: 「生成」ボタン、履歴表示
- **モバイル最適化**: タッチフレンドリーなインターフェース

## 4. Web画面 UI 全体設計

### 4.1 全体レイアウト
- **レスポンシブ**: iPhone 中心、横幅 375px-414px
- **ナビゲーション**: ボトムタブ（着せ替え、生成、設定）
- **カラムベース**: 機能に応じて 1-3 カラム切り替え

### 4.2 コンポーネント
- **ヘッダー**: タイトルとメニューアイコン
- **メインエリア**: キャンバスとコントロール
- **フッター**: アクションボタン

### 4.3 デザイン指針
- **直感性**: ドラッグ&ドロップで画像入力
- **パフォーマンス**: ローディングアニメーションで待機時間をカバー
- **アクセシビリティ**: 高コントラスト、音声読み上げ対応

### 4.4 画面遷移
- **ホーム**: 機能選択
- **着せ替え**: 2カラム入力
- **生成**: キャンバス中心
- **結果**: フルスクリーン表示
