# NpureStudio 概要設計書（更新）
## 1. プロジェクト概要
NpureStudio は iPhone の NPU と WebGPU を活用し、端末上で日本語から高速に画像を生成・編集するブラウザアプリケーションです。
## 2. 解決する課題
- 言語の壁を取り除き、日本語だけで高品質な画像生成体験を提供する。
- 生成・編集の処理を端末内で完結させ、プライバシーと低遅延を両立する。
## 3. 主要機能（優先順位）
1. 日本語 Text-to-Image（最優先）
    - 日本語プロンプトを軽量な翻訳モデルで英訳し、WebGPU 上の生成モデルで画像を作成します。
2. 生成ベースのインペイント／着せ替え（応用）
    - 生成画像や既存の人物写真に対して、日本語で部分編集（インペイント）を行います。
3. マルチキャンバス編集
    - 生成した画像をキャンバス上で配置・合成・微調整できるエディタ機能。
## 4. 技術スタック（概要）
- Runtime: WebGPU（Safari）
- Engine: Transformers.js v3（WebGPU/WASM）
- Translation: 軽量 T5 系モデル（日本語→英語）
- Generation: ONNX 化した Stable Diffusion 系（量子化・FP16 推奨）
- Deployment: GitHub Pages（静的配信）
## 5. モデル配布ポリシー
- クライアントが公開 HF Hub または CORS 対応 CDN からモデルを取得してキャッシュする方式を基本とします。
- 非公開モデルは署名付き短命 URL を用いた配布を推奨します（ブラウザに長期トークンを置かない）。
## 6. 開発フェーズ（短期優先）
- Phase 1: WebGPU と Transformers.js の環境整備（完了）
- Phase 2: 日本語翻訳（translation.js）と Text-to-Image（generation.js）のプロトタイプ
- Phase 3: 生成／インペイントの UX 統合（着せ替えは応用フェーズ）
詳細は `docs/詳細設計.md` を参照してください。
# NpureStudio 概要設計書

## 1. プロジェクト概要
iPhoneのNPU（Neural Processing Unit）とWebGPU技術を活用し、Safariブラウザ上で完結する「オンデバイスAI画像生成・編集プラットフォーム」を構築する。

## 2. 解決したい課題
- 既存アプリ（Draw Things等）における「単一キャンバス」「画像入力数の制限」といったUIの不自由さを解消する。
- クラウドサーバーを介さず、プライバシーを保護しながら爆速でAI編集を行う。

## 3. 主要機能 (Core Features)
### A. 自動着せ替え（Virtual Try-On）
- **入力:** 人物写真、衣服写真の2点。
- **処理:** 1. AIによる人物・衣服の自動セグメンテーション（マスク生成）。
    2. IP-Adapter等を用いた衣服の特徴抽出。
    3. インペイント技術による衣服の差し替え。
### B. 汎用画像生成・編集
- **Text-to-Image:** プロンプトからの画像生成（SDXL Turbo等）。
- **Image-to-Image:** 写真のスタイル転送、背景チェンジ。
- **Object Removal:** 不要なものの削除。

## 4. 技術スタック
- **Runtime:** WebGPU (Safari 26+)
- **Engine:** Transformers.js v3
- **Framework:** Vanilla JS or React (TBD)
- **Deployment:** GitHub Pages

## モデル配置ポリシー
- **ローカル非配置:** 本プロジェクトではモデルをリポジトリや公開サーバの `models/` に配置しません。クライアントは公開された Hugging Face Hub（CORS 対応）または CORS 対応の CDN から直接ダウンロードしてキャッシュする方式を原則とします。
- **非公開モデル:** 非公開モデルを利用する場合は短命の署名付き URL を発行する最小限のプロキシサーバを使う運用を推奨します（ブラウザにトークンを埋め込むのは避ける）。

## 5. 開発フェーズ (Milestones)
- [x] Phase 1: WebGPU 疎通確認テスト（完了）
- [x] Phase 2: Transformers.js 導入と基本画像認識テスト（完了）
- [x] Phase 3: 自動セグメンテーション（SAM）の実装（完了）
- [ ] Phase 4: Stable Diffusion (Inpainting) による試着機能実装
- [ ] Phase 5: マルチキャンバスUIの構築

## 6. UI/UX 指針
- 「左に自分、右に服」といった、直感的な2カラム以上のレイアウト。
- iPhoneのタッチ操作に最適化したキャンバス操作（ピンチイン・アウト、マスク微調整）。
